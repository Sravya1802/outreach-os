--------------------------------------------------------------------------------
PHASE 0 - Preparation (my work, ~2 hours)
--------------------------------------------------------------------------------

Before you touch Oracle, I prepare:

1. deploy/oracle-setup.sh - the one-shot installer script
   What it does automatically on your VM:
   - apt update && apt upgrade
   - Install Node 20, nginx, certbot, pm2, Playwright OS deps
   - Clone the repo
   - npm install in backend + frontend
   - Install Playwright Chromium binary
   - Configure nginx as reverse proxy (80/443 -> 3001)
   - Obtain Let's Encrypt SSL via DuckDNS subdomain
   - Start backend under pm2 with auto-restart + boot-time launch
   - Install keep-alive cron (prevents Oracle idle reclamation)
   - Enable unattended-upgrades (auto security patches)
   - Install fail2ban (SSH brute-force protection)

2. deploy/env.template - env var file you'll paste keys into

3. backend/ecosystem.config.cjs - pm2 config

4. backend/nginx.conf - nginx server block

5. Dockerfile fallback - in case we ever switch container hosts

    sudo ./setup.sh

    - Path to env file (I give you a template, you paste API keys)

    Runs ~25 min unattended. You watch log.

PHASE 4 - sqlite -> Supabase migration via pg (6-10 hrs my work, 1-2 sessions)
--------------------------------------------------------------------------------

4a - Connection + schema parity (2 hrs)
    - Add pg package to backend
    - New backend/db.js using pg.Pool connected to Supabase
    - Generate Postgres schema from sqlite CREATE TABLE statements
    - Write supabase/migrations/002_backend_tables.sql
    - Run it via Supabase SQL editor

4b - Query rewriting (4-6 hrs)
    - Find/replace db.prepare(...).get(...) -> await pool.query(...).then(r => r.rows[0])
    - Translate sqlite dialect: INSERT OR IGNORE, datetime('now'), LAST_INSERT_ROWID
    - Async-ify every route that touches the DB
    - Preserve transactions where needed

4c - Data backfill (30 min, optional)
    - Export sqlite from outreach-local to CSV
    - Import into Supabase via COPY
    - Or skip and start fresh on prod

4d - Per-route smoke test (1-2 hrs)
    - curl every endpoint, verify response shape unchanged
    - Check Supabase writes happen correctly

FALLBACK if migration blocks:
    Keep sqlite on VM disk (works, persists as long as VM lives).
